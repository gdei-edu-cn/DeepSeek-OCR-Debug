## Case 1：数字表格被 vt64 幻觉放大（en_73）

- **页面**：`en_73.png`
- **错误类型**：
  - vt64：number[ins]（大量数字插入）
  - vt100：number[sub]（少量数字替换）
- **GT 片段**（来自 vt100 样例）：
  - `..., 57, 86, 154, ...`
- **vt64 行为**（多条 number[ins]）：
  - `PRED: ... 332, ...`
  - `PRED: ... 1673, ...`
  - `PRED: ... 1426, ...`
  - `PRED: ... 1248, ...`
  - `PRED: ... 563, ...`
  - `PRED: ... 257, ...`
  - `PRED: ... 128, ...`
  - `PRED: ... 183, ...`
  - `PRED: ... 1211, ...`
  - `PRED: ... 1325, ...`
  - `PRED: ... 1048, ...`
- **vt100 行为**：
  - `GT  : ... 57, 86, 154, ...`
  - `PRED: ... 174, ...`（把一串数字缩成单个 174）
- **现象说明**：
  - 在同一页的数字密集区域，vt64 会“凭空”加出大量不存在的数字，形成严重的数值幻觉；
  - vt100 仍然会在个别位置改错数字，但规模小得多。
- **语义影响**（可以后续补充）：
  - 表格/数据列表被大量虚构数字污染，整页统计结论可能完全失真；
  - 这是 Fox-100 上 vt64 的典型灾难性错误模式。

## Case 2：投票/计数被压缩得很离谱（en_16）

- **页面**：`en_16.png`
- **错误类型**：
  - vt64：number[sub] 和 math_symbol[sub]
- **错误示例 1（数字）**：
  - `GT  : ... abstentions. 300. The ...`
  - `PRED: ... 40. ...`
- **错误示例 2（符号 / 词形）**：
  - `GT  : ... AND NON-DISCRIMINATION IN ...`
  - `PRED: ... NONDISCRIMINATION ...`
- **现象说明**：
  - vt64 把“300 票弃权”读成了 “40”，数量级直接错了一个数量级；
  - 同一页上还把包含连字符的长词压扁成一个 token，丢失了 AND / 连字符等语法信息。
- **语义影响**：
  - 对“投票结果 / 统计数字”这类句子来说，300→40 属于非常关键的语义错误；
  - 这类错误会直接改变文本表达的结论。

## Case 3：日期和年份被破坏（en_19）

- **页面**：`en_19.png`
- **错误类型**：
  - vt64：number[ins], number[del]
  - vt100：number[del]
- **vt64 示例**：
  - 插入额外年份：
    - `PRED: ... 2013 ...`（GT 对应位置为空）
  - 丢掉日期中的年份：
    - `GT  : ... June 30, 2016. ...`
    - `PRED: ...  ...`（`2016` 被删掉）
- **vt100 示例**：
  - `GT  : ... 1, 2015 and ...`
  - `PRED: ...  ...`
  - `GT  : ... Year 2015 begins ...`
  - `PRED: ...  ...`
- **现象说明**：
  - 两个模式都说明模型在处理日期/年份时容易出错；
  - vt64 既会乱加年份，又会在关键日期位置把年份删掉；
  - vt100 错误总体少得多，但仍然会丢掉年份信息。
- **语义影响**：
  - 对法律/财务文本，年份往往非常关键，错误年份会导致引用条款或时间范围完全不同。

## Case 4：法律条文标号被打乱（en_97）

- **页面**：`en_97.png`
- **错误类型**：
  - vt64：math_symbol[sub] / math_symbol[del]
- **示例 1：替换**：
  - `GT  : ... §11.3, 487.17-18 and ...`
  - `PRED: ... "guru ...`
- **示例 2：删除**：
  - `GT  : ... §11.3, 487.10-17: ayam ...`
  - `PRED: ...  ...`
- **现象说明**：
  - 这里 GT 是标准的法律条文引用（条款号 + 页码范围）；
  - vt64 要么把整个引用替换成完全不相关的 token（"guru），要么直接删掉；
  - 对于依赖精确引用的法律/学术文档，这是非常严重的错误。
- **语义影响**：
  - 读者无法根据文本找到对应条款；
  - 这类错误在 CER 里只是几个 token，但真实影响非常大。

## Case 5：参考号 / 索引被批量幻觉（en_88, en_95）

- **页面**：`en_88.png`, `en_95.png`
- **错误类型**：
  - vt64：math_symbol[ins]
- **en_88 示例**：
  - `GT  : ...    ...`
  - `PRED: ... 2258-2259, ...`
  - `PRED: ... 2201-2202, ...`
- **en_95 示例**：
  - `GT  : ...    ...`
  - `PRED: ... (22-04-1086) ...`
- **现象说明**：
  - 这些看起来像是卷宗号 / 条目号 / 时间戳一类的复杂编号；
  - 在 GT 对应位置是空的，说明 vt64 完全“幻觉”出了这些编号；
  - 这类错误在 word-level 看似只是多写了一串字符，但对检索/引用非常致命。
- **语义影响**：
  - 读者可能会误以为这些编号对应真实的法条或文献；
  - 在检索系统中会引入大量虚假条目。

## Case 6：vt100 的典型残余错误（en_25, en_80, en_40 等）

- **页面**：`en_25.png`, `en_80.png`, `en_40.png` 等
- **错误类型**：
  - vt100：number[sub], math_symbol[sub]
- **示例 1：词形/拼写类（影响较小）**：
  - `GT  : ... over time-spans, equivalent ...`
  - `PRED: ... time-spends, ...`
  - `GT  : ... a branchand-bound strategy ...`
  - `PRED: ... branch-and-bound ...`
- **示例 2：数字替换（局部但关键）**：
  - `GT  : ... vocabularies. 3 There ...`
  - `PRED: ... list ...`（数字变成普通单词）
  - `GT  : ... approximately sixty-six billion ...`
  - `PRED: ... sixty-seven ...`（数量级完全不同）
  - `GT  : ... 20. 10 Robertson, ...`
  - `PRED: ... 5 ...`
- **现象说明**：
  - 大多数 vt100 的错误是局部的数字替换或脚注号丢失，相比 vt64 的整段幻觉要温和很多；
  - 但个别情况下（如 “sixteen billion”→“sixty-seven”）仍然会改变关键数值。
- **语义影响**：
  - 总体来说，vt100 的错误更像是“精度不够”而不是“整页崩溃”；
  - 这些例子可以在论文中用来说明：即使在 4% CER 下，仍然存在高风险的数值错误，需要进一步控制。
